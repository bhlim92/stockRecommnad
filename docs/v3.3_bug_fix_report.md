# v3.3 UI 무응답 버그 수정 보고서

## 1. 개요
- **장애 일시**: 2026-06-07
- **장애 현상**: 퀀트 스크리너(`screener.html`) 화면에서 상단의 시장 변경 탭(KOSPI 200, KOSDAQ 등) 및 '퀀트 스캔 시작' 버튼을 클릭해도 아무런 반응(초록색 활성화, 데이터 페칭 등)이 일어나지 않는 현상 발생.
- **영향 범위**: `index.html` 및 `screener.html`을 포함한 전체 프론트엔드 UI 인터랙션 마비.

## 2. 원인 분석
클라이언트 사이드에서 버전을 동적으로 가져오기 위해 삽입된 **Dynamic Version Fetcher** 스크립트가 근본 원인이었습니다.

```javascript
// 문제의 코드
fetch('/api/version')
    .then(res => res.json())
    .then(data => {
        const version = data.version || '3.3';
        document.body.innerHTML = document.body.innerHTML.replace(/\{\{VERSION\}\}/g, version);
    })
```

- **DOM 파괴 및 재생성**: `document.body.innerHTML`을 읽어서 문자열 치환 후 다시 할당하는 과정에서, 브라우저는 기존의 모든 HTML 요소를 파괴하고 완전히 새로운 DOM 요소로 재생성합니다.
- **이벤트 리스너 유실**: 이로 인해 페이지 로드 시점(`DOMContentLoaded`)에 각 버튼과 탭 요소에 바인딩되었던 Javascript Event Listener(`addEventListener`)들이 모조리 메모리상에서 연결이 끊어져(Orphaned), 클릭 이벤트가 아예 감지되지 않는 '죽은 UI' 상태가 되었습니다.

## 3. 해결 방안 및 조치 내역
### 조치 내역
1. **불필요한 클라이언트 사이드 스크립트 삭제**: Vercel이 서버리스 함수(`app.web_server.render_html_with_version`)를 통해 HTML 서빙 시 이미 서버단에서 `{{VERSION}}` 치환을 완료하여 내려주므로, 클라이언트에서 다시 버전을 fetch하여 DOM을 건드릴 이유가 전혀 없습니다.
2. `app/static/index.html` 및 `app/static/screener.html`에서 해당 스크립트 블록을 완전히 제거.
3. Vercel CLI를 통해 수동 강제 배포(`npx vercel --prod --yes`)를 진행하여 막혀있던 CI/CD 캐시 갱신 완료.

### 결과
- DOM 파괴가 발생하지 않아 `screener.html`의 모든 이벤트 리스너가 정상 유지됨.
- 시장 탭 클릭 시 정상적으로 KOSPI, KOSDAQ 시장 데이터 연동 확인.
- 스캔 시작 버튼 클릭 시 프로그레스 로딩 및 백엔드 스캔 구동 정상 확인.
